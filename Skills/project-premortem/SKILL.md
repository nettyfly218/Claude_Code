# 预演失败 (Project Premortem)

此技能用于引导用户进行"事前验尸"（Pre-mortem）练习。不同于传统的风险评估，我们要求用户和AI完全站在"项目已经失败"的未来时间点，回溯导致死亡的路径。

## 核心流程

### 第一步：设定失败场景
**必须**以确定的口吻宣告项目失败。
示例话术："现在是项目启动后的第18个月。很遗憾地通知你，[项目名称]已经正式停止运营/被判定彻底失败。团队解散，投资归零。让我们站在废墟上，诚实地回答发生了什么。"

### 第二步：深度拷问 (The 4 Questions)
结合用户提供的项目背景，详细回答以下四个问题。回答必须尖锐、具体，避免由于礼貌而含糊其辞。

1.  **衰退信号 (The Decline)**
    *   *什么时间点开始出现衰退信号？*
    *   描述具体的指标异常、团队氛围变化或市场反馈的转折点。

2.  **致命决策 (The Fatal Error)**
    *   *最致命的决策错误是什么？*
    *   指出这期间做出的一个看起来合理但实际上毁灭性的决定（例如：过早扩张、选错技术栈、忽视某类用户声音）。

3.  **核心盲区 (The Ignored Risk)**
    *   *忽视的核心风险是什么？*
    *   指出那个一直在房间里但大家都不愿意讨论的"大象"。

4.  **重启修正 (The First Fix)**
    *   *如果能重来，第一个应该改的是什么？*
    *   具体到第一周甚至第一天的一个动作。

### 第三步：寻找真实镜像
使用搜索工具寻找 **1-2个现实世界中性质相似但失败了的真实项目案例**。
*   分析它们的失败轨迹是否与上述推演吻合。
*   提取它们的教训作为旁证。

### 第四步：撰写《失败复盘文章》
基于上述分析，以第三方深度科技媒体（如"晚点LatePost"或"36氪"深度报道风格）的口吻，写一篇复盘文章。

**文章结构要求：**
*   **标题**：具有警示性与冲击力（例如：《[项目名] 败局：一场由于傲慢引发的雪崩》）。
*   **导语**：沉重而冷静地叙述结局。
*   **正文**：融合第二步的四个问答和第三步的真实案例对比。分析要入木三分。
*   **结语**：留给后来者的墓志铭。

## 注意事项
*   **保持悲观**：在此模式下，严禁盲目乐观。必须假设最坏的情况发生。
*   **基于人性**：失败往往源于人性的弱点（贪婪、恐惧、懒惰、傲慢），分析时要触及这个层面。
