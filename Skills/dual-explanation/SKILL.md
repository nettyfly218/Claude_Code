# 双层解释法 (Two-Layer Explanation)

这个技能用于以两种截然不同的风格解释概念：
1. **初学者版本（大爷版）**：使用极度通俗、生活化的类比（通常是洗脚城、菜市场等生活场景），确保没有任何技术背景的老大爷也能听懂。
2. **深度专业版本（专家版）**：使用学术、严谨的专业术语，引用底层原理，确保内容的准确性和深度。

## 使用指南

当用户询问一个概念或问题时，请严格按照以下格式输出：

### 1. 👴 初学者版本（大爷版）

*   **核心心法**：把所有技术名词替换成生活中的大白话。
*   **场景设定**：洗脚城、麻将馆、菜市场、公园下棋。
*   **语气**：亲切、接地气、甚至带点江湖气。
*   **重点**：用大爷能听懂的“人话”解释复杂逻辑，多用比喻。

### 2. 🧑‍💻 深度专业版本（专家版）

*   **核心心法**：精准、全面、深入底层。
*   **要求**：绝对严谨，不能出现事实错误，使用标准专业术语。
*   **内容**：涵盖定义、核心原理、架构设计、优缺点分析、底层实现细节等。

## 示例

**用户输入**：什么是 面向对象 (OOP)？

**输出**：

### 1. 👴 初学者版本（洗脚城大爷版）

大爷，您坐好，我给您捏捏脚。这“面向对象”啊，就好比咱们这洗脚城的**VIP会员制度**。

您看以前咱们办事（面向过程），那是“先打水，再脱鞋，再捏脚，再擦干”，一步步来，换个技师可能就不会了。

现在这“面向对象”不一样：
*   **封装**：咱们把“捏脚”这一套手艺，打包教给技师（对象）。您不用管技师是怎么用力的，您只要喊一声“技师，上钟！”，活儿就干了。
*   **继承**：新来的技师小王，是他师父老张带出来的。老张会的推拿，小王天生就会，这就叫继承。小王还能再去学个拔罐，这叫扩展。
*   **多态**：您喊一句“技师，加个钟！”，这老张可能是给您按背，小王可能是给您按头。同一个指令，不同的人干出不同的活儿，这就叫多态。

所以啊，面向对象就是把活儿分给一个个专业的人（对象）去干，省心！

### 2. 🧑‍💻 深度专业版本（专家版）

面向对象编程 (Object-Oriented Programming, OOP) 是一种基于“对象”概念的编程范式，对象包含数据（属性）和代码（方法）。

*   **核心概念**：
    *   **封装 (Encapsulation)**：将数据和操作数据的方法绑定在一起，并隐藏对象的内部实现细节，只暴露必要的接口。这提高了安全性和模块化程度（如 Java 中的 private/public 访问修饰符）。
    *   **继承 (Inheritance)**：允许创建基于现有类的新类，复用代码并建立层级关系。子类继承父类的属性和方法，体现了 "is-a" 的关系。
    *   **多态 (Polymorphism)**：允许不同类的对象对同一消息作出响应。实现方式包括重载 (Overloading) 和重写 (Overriding)。运行时多态通常通过虚函数表 (vtable) 实现。
*   **设计原则 (SOLID)**：
    *   单一职责原则 (SRP)
    *   开闭原则 (OCP)
    *   里氏替换原则 (LSP)
    *   接口隔离原则 (ISP)
    *   依赖倒置原则 (DIP)
*   **优势**：提高了代码的重用性、灵活性和可维护性，特别适合大型复杂系统的开发。
