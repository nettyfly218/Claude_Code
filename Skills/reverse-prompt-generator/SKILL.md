# Reverse Prompt Generator Instructions

当用户提供一段文本或文档并要求生成对应的Prompt时，请执行以下步骤：

## 1. 深度分析 (Analyze)

仔细阅读用户提供的文本，分析以下维度：
- **核心主题 (Core Subject)**: 文本在说什么？
- **角色设定 (Persona)**: 作者看起来是什么身份？（如：资深工程师、甚至的小说家、严谨的学者）
- **语气与风格 (Tone & Style)**: 幽默、严肃、口语化、学术、抒情等。
- **结构与格式 (Structure & Format)**: 分段方式、标点习惯、是否使用Markdown、列表、特定标题格式等。
- **特殊约束 (Constraints)**: 字数限制、必须包含的元素、禁止出现的元素。

## 2. 编写提示词 (Draft Prompt)

基于分析结果，编写一个结构化的Prompt。建议包含以下模块：
- **Role (角色)**: 定义AI的身份。
- **Context (背景)**: 设定任务背景。
- **Task (任务)**: 明确具体的生成指令。
- **Constraints (约束)**: 规定风格、格式、长度等限制。
- **Workflow (工作流)**: 如果任务复杂，定义Step-by-step的思考过程。
- **Examples (示例)**: (可选) 将用户提供的文本作为Few-Shot示例放入Prompt中。

**Prompt 输出格式**: 请使用 Markdown 代码块包裹生成的 Prompt。

## 3. 解析说明 (Explain)

在生成 Prompt 后，逐条解释 Prompt 中关键句子的作用。
格式如下：
- **[引用 Prompt 中的某一句/段落]**: 解释为什么这么写，它如何帮助还原原文的风格或逻辑。
